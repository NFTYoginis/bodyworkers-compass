# Architecture — one slice now, a frame built to grow

This public researcher is **one entry into a larger frame.** It carries a single
region — chronic low-back / posterior chain — fully worked at sampler depth. The
architecture is deliberately built so additional regions slot in without
re-designing anything. This file documents the frame so the shape is visible even
though the deeper content is not public.

---

## What's public vs. what's gated

| Layer | Public (in this repo) | Gated (operator-local / private tool) |
|---|---|---|
| **Method** | Evidence tiers, the governor, register discipline, the vocabulary bridge — all public research | — (the method is fully public) |
| **Region content** | The ONE low-back slice at sampler depth | The full multi-region atlas |
| **JDM** | The investigative spine + the named vocabulary, at sampler depth | JDM internals beyond the one demonstrated pass |
| **Sen lines** | Names + public physical territory + "defer to source" | All lineage internals (operator / teacher is the source) |
| **Decks / books** | Funnel links only (`gabeyoga.com/cards`, `/books`) | Anything verbatim from the four books or the saleable deck PDFs |

The boundary rule is simple: **the method is fully public; the depth is gated.**
The public slice proves the reusable skill; the depth is the product the funnel
points to.

---

## The frame (how a region is shaped)

Every region — public or private — takes the **same six-lens shape** the low-back
slice demonstrates:

1. **Joint / skeletal-muscular anatomy** — Tier 1 biomechanics for the region.
2. **Anatomy Trains** — which myofascial line(s), each tier-tagged (confirmed vs.
   unsupported — *always name which*).
3. **TCM / meridian** — which channel(s); the Langevin location-correlation tier;
   Cooley's emotional layer flagged as inquiry-only.
4. **Thai Sen lines** — which line(s); named + physical territory; depth deferred
   to source.
5. **The JDM vocabulary bridge** — the finding carried across the maps in
   register-neutral terms (Short/Long, Cup, Four Words, Tent).
6. **Convergence points** — named single-point overlaps (the GB-30 kind),
   presented as *location* convergence only.

Because every region is this shape, the researcher's *method* transfers
unchanged; only the region's content fills in.

## The gated `_atlas/` — home for the private partner tool

The real prize the public slice points to is a larger **private partner tool**
backed by the four books and the supporting material, for the bodyworker world.
It is not built here. But the frame anticipates it:

- `_atlas/` (underscore-prefixed, **gitignored, absent from the public repo**) is
  the extensible home for the private tool's multi-region content.
- Each new region drops in as a sibling folder using the six-lens shape above.
- The public low-back slice is the *template* every private region copies.

> Don't build the private tool in the public repo. The public repo demonstrates
> the method and funnels to the products; the partner tool grows in `_atlas/`,
> region by region, where the gated depth lives.

When the partner tool is ready to expand, the operator adds regions under
`_atlas/<region-slug>/` (see `_atlas/README.md` in the local clone) — neck,
shoulder girdle, anterior chain, deep front line / core, arm lines, and so on —
each one built on exactly the shape this low-back slice already proves.

---

## Why this matters for the build's credibility

The architecture *is* part of the honesty. By keeping the method public and the
depth gated — and by saying so plainly — the researcher avoids the two failure
modes that sink bodywork tools:

- **Over-exposure:** dumping the full atlas / lineage internals for free, which
  both devalues the products and risks authoring claims that aren't the builder's
  to make.
- **Over-claiming:** pretending the one public slice is the whole validated
  system. It isn't, and the tool says so.

One honest slice, a frame built to grow, and a clear line between what's
demonstrated and what's deferred. That's the architecture.
